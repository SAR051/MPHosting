# MC Watchdog — موقع لمراقبة سيرفر ماينكرافت وتشغيل بوت دخول (مثال)

ملخص
- واجهة بسيطة لإضافة سيرفر (IP + port).
- أي شخص يستطيع الضغط "Join" ليجعل بوت (mineflayer) يدخل السيرفر لمدة محددة (افتراضي 8 ساعات).
- خيار "Save" يجعل النظام يراقب السيرفر تلقائياً ويشغّل البوت عند رجوعه Online.
- لا يتحكم بالمضيف (Aternos) لبدء السيرفر — فقط يتصل بالـ IP:PORT عندما يصبح متاحًا.

متطلبات
- Node.js 18+، npm
- Redis (موصى به) — يمكنك تشغيله محليًا أو بواسطة Docker.

تشغيل محلي (بدون Docker)
1. انسخ الملفات أعلاه إلى مجلد.
2. انسخ `.env.example` إلى `.env` وعدّل إذا لزم.
3. ثبّت الحزم:
   npm install
4. شغّل Redis (مثلاً عبر docker run -p 6379:6379 redis:7-alpine) أو استخدم خادم Redis موجود.
5. شغّل التطبيق والـ worker (في نافذتين):
   - نافذة 1: npm run start
   - نافذة 2: npm run worker
6. افتح http://localhost:3000 وأضف سيرفرك (مثلاً Itabbd.aternos.me:48729)، اضغط Save إذا تريد المراقبة التلقائية.
7. اضغط "Join" ليجعل البوت يدخل فوراً (إذا كان السيرفر Online).

تشغيل عبر Docker Compose
1. ضع الملفات، أنشئ `.env` إذا أردت.
2. شغّل:
   docker compose up --build

ملاحظات أمنية وقانونية
- احصل على إذن من صاحب السيرفر قبل تشغيل بوت.
- إذا السيرفر يتطلب مصادقة Microsoft، يحتاج حل أمنّي إضافي (هذا المثال يدعم offline mode فقط).
- لا تحفظ أو تشارك بيانات الحسابات عبر هذا المشروع.

خطوة مقبلة (اختياري)
- إذا تريد أضيف آلية لبدء السيرفر على Aternos (الضغط على Start تلقائياً)، زودني برابط صفحة Aternos وسأضيف وحدة Playwright مع persistent profile وشرح كيفية تسجيل الدخول الآمن. هذا يتطلب الحذر لأن الأتمتة قد تخالف شروط Aternos.
